Design


Transmission Control Protocol

i vores program har vi transmission control protocol, som også kan forkortes til TCP.

  • TCP er en "pålidelig" protokol, med andre ord, det garanterer levering af data den sender i form af "pakker."

  • Hvis pakkerne bliver tabt eller beskadiget, vil TCP gensende dataen, indtil det bekræfter, at pakkerne er blevet overført.

  • TCP skal være der i Applicationer, som pålideligt skal sende beskeder frem og tilbage.


TCP/IP Sockets

Når forbindelsen er blevet skabt, har Client og Server begge en binding til hver deres sockets,i deres forbindelse. Når en forbindelse er etableret kan Client og Server både læse fra og skrive "socket" når der kommunikeres.


Client-Server Application

  1. Opret en Socket og ServerSocket objekt og åben det over en specificeret "port" og "ip" adresse eller hostsnavn. Eksempel - "localhost", 9090

  2. Instantiere "InputStreamReader, BufferedReader og PrintStream" objekter til at tilsætte data til og fra hver socket.

  3. læse og skrive fra stream mellem sockets ved at bruge det aftalte protocol.

  4. lukke input og output streams

  5. lukke Sockets eller ServerSocket objects.


State Behaviour

den "Server" som er skabt af ServerSocket object og åben den over en specifik "port" og "ip" addresse eller host navn, "localhost", 9090. clienten opretter en socket, og når en forbindelse mellem sockets er blevet skabt, vil clienten og server begge læse fra og skrive til socketen. Serveren kalder the "accept()" method og skaber en ny Thread for clienten. når "Terminalen" bliver brugt på Mac eller "Command Prompt" på Windows, det første du skal gøre er bruge protocol tag - "USER#name", til at skabe en user. når en user er blevet skabt, vil "USERLIST#" blive tilgængelig med alle online/connected users. Alle online/connected clients vil modtage listen. kommunication mellem clients, er lavet gennem den aftalte procotol - "MESSAGE#receiver#msg".


Observer Design Pattern

vi har implementeret Observer design pattern. The Observer design pattern definerer en one-to-many dependency mellem objects så at når en object ændre tilstand, will alle dens dependents være notified og automatisk updated. Observer design pattern falder under "behavioral pattern" katagory. vores "GUI" class implementere observer, med en abstract method "update()" implemented. "Clienten" class extends "Observable", og hvergang vi får en message, vil "notfiy()" method blive kaldt . "notify()" method notifieciere til alle the observers.
